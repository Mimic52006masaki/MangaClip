# 06_NonFunctionalRequirements.md

## パフォーマンス
- 想定同時接続数：1〜2人（個人利用）
- レスポンスタイム目標：記事取得・コピー処理は1秒以内、スクレイピングは10秒以内

## セキュリティ
- 認証方式：APIキー保護（個人利用）
- データ保護方針：DBアクセス権限管理（SQLiteの場合はファイル権限管理、クラウドDBの場合はユーザー認証設定）

## 可用性・運用
- 障害時の対応：スクレイピング失敗時はエラーメッセージ表示
- ログ・監視：スクレイピング日時と取得件数ログのみ保持
- バックアップ：DBデータはJSON/CSV形式で定期バックアップ可能
