# アプリ要件定義

## 1. 目的・背景
- このアプリで解決したい課題：
  - 漫画・アニメまとめサイトの記事管理を効率化する
  - 手動で記事タイトルやURLをコピー・整理する手間を削減する
- なぜ今このアプリが必要か：
  - 取得件数が増え、スプレッドシート管理が煩雑になっている
  - SEO対策用タイトル生成など自動化したい作業がある

## 2. ゴール
- このアプリが「成功」と判断できる状態：
  - 漫画まとめ速報の記事は日付・時間ごとに整理され、1日単位でURLを一括操作可能
  - アニメまとめCHの記事はタイトルとURLを簡単にコピーできる
  - 重複取得を防ぎ、バックアップ可能

## 3. 想定ユーザー
- 主なユーザー：
  - 個人のコンテンツ管理者
- 利用シーン：
  - 記事取得後の整理・コピー・Web参照

## 4. 機能一覧
### Must（必須）
- 記事取得（漫画まとめ速報・アニメまとめCH）
- 漫画まとめ速報：targetDate自動割り当て（07:00〜21:00）
- 漫画まとめ速報：1日単位URL一括開き
- 漫画まとめ速報：生成タイトル1件ずつコピー
- 漫画まとめ速報：レコード削除
- アニメまとめCH：タイトル・URL全件コピー
- データ重複防止
- バックアップ機能

### Nice to Have（あれば良い）
- 生成AIによるSEOタイトル自動生成
- 取得対象サイトの追加（拡張性）

## 5. 対応プラットフォーム
- Web / Windows / Mac（Electronやブラウザベースを想定）

## 6. 対象外（やらないこと）
- リアルタイム通知
- マルチユーザー同時編集
- 高度な分析やレコメンド機能
